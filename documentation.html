2. Project Overview

Purpose

MedConnect is engineered to address scheduling friction, record safety, and verification opacity in healthcare discovery. For patients, it provides an intuitive discovery search directory with filtering parameters, real-time booking availability, and secure clinical records locker. For medical practitioners, it offers a workspace to onboard credentials, configure calendar hours, audit patient medical histories, and manage consultation workflow requests. For administrators, it offers dashboard toggles to review registrations, moderate database logs, and inspect platform metrics.

Core Application Features

  • User Identity & Verification: JWT credentials authentication mapping custom onboarding roles (Patient, Doctor, Admin) with email confirmation codes fallbacks.
  • Specialist Discovery Directory: Instant doctors lookup engine filtering search matches by specialty, clinic location, experience years, fee ranges, and sorting preferences.
  • Anti-Collision Scheduling: Appointment booking calendar with compound unique index constraints in Mongoose to prevent double-booking slot conflicts.
  • HIPAA-Safe Records Sharing: Secure patient PDF/image uploads, restricted strictly to consulting doctors with active appointment history logs.
  • Socket.io Notifications: Real-time push updates displaying browser alerts upon confirmations, completes, or cancellation events.
  • Aggregate Feedback Analytics: Automatic re-calculation of average ratings and total reviews counts on the Doctor document on patient feedback submissions.

4. Setup Instructions

Prerequisites

  • Node.js v16 or above (Recommended: v18 LTS)
  • MongoDB Community Server locally running on port 27017

Installation

1. Configure environmental secrets: Create a .env file under the server directory:

PORT=5000
MONGO_URI=mongodb://localhost:27017/medconnect
JWT_SECRET=medconnect_jwt_super_secret_key_123456
CLOUDINARY_NAME=
CLOUDINARY_KEY=
CLOUDINARY_SECRET=
EMAIL_USER=
EMAIL_PASS=

2. Install backend server dependencies:

cd backend
$env:PATH = "C:\Program Files\nodejs;" + $env:PATH
npm install

3. Install frontend client dependencies:

cd ../frontend
$env:PATH = "C:\Program Files\nodejs;" + $env:PATH
npm install

8. Authentication

MedConnect uses stateless JSON Web Token (JWT) credentials mapping. The token is generated during logins/registrations and signed with a JWT secret key, then forwarded via header Authorization Bearer tokens. 

export const protect = async (req, res, next) => {
  let token;
  if (req.headers.authorization && req.headers.authorization.startsWith('Bearer')) {
    token = req.headers.authorization.split(' ')[1];
  }
  if (!token) return res.status(401).json({ message: 'Not authorized, token missing' });
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = await User.findById(decoded.id).select('-password');
    next();
  } catch (error) {
    return res.status(401).json({ message: 'Token invalid' });
  }
};
